Metadata and Catalogue

In this page we provide both ODOHTEU catalogue for the datasets we have created and thirteen tables, one for each dataset, reporting metadata in a human-readable way; since chose to include metadata for the entire catalogue, but also separately for each dataset. We provided data with its metadata in order to provide effective reusable and interoperable data, adopting the DCAT AP version 2.0.0 specification and we chose Turtle as the serialization.

The need for metadata in rdf derives from the goal of developing a project that was compliant with LOD characteristics: from the use of open and machine-readable formats for the datasets to the specification of the license. Following the 5-STAR OPEN DATA MODEL, we have used meaningful, persistent, dereferenceable and unambiguous; as well as supported by a reliable infrastructure
